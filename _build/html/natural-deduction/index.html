5.5 Reduction to the absurd. P⇒Q, ¬Q⊢¬PP \Rightarrow Q,\ ¬ Q ⊢ ¬ PP⇒Q, ¬Q⊢¬P¶

This is a very useful technique. Validity of P⇒Q, ¬Q⊢¬PP \Rightarrow Q,\ ¬ Q ⊢ ¬ PP⇒Q, ¬Q⊢¬P is proved with:

The sub-derivation (lines 3–5) discharges the hypothesis PPP: assuming PPP, [⇒\Rightarrow⇒-elimination] on lines 1 and 3 gives QQQ, while line 2 reiterates ¬Q¬ Q¬Q. The contradiction lets us introduce ¬P¬ P¬P (¬¬¬-introduction over lines 3, 4, 5).

In Lean this is theorem T55, where ¬P is definitionally P → False,[1] so the whole proof is just function composition:

theorem T55 (h1 : P → Q) (h2 : ¬Q) : ¬P :=
  (h2 ∘ h1)

Conjunction introduction. P, P⇒Q⊢P∧QP,\ P \Rightarrow Q ⊢ P \wedge QP, P⇒Q⊢P∧Q¶

Theorem T51 builds the pair ⟨proof of P, proof of Q⟩ directly, using the anonymous-constructor notation ⟨_, _⟩:

theorem T51 (h1 : P) (h2 : P → Q) : P ∧ Q :=
  ⟨h1, h2 h1⟩
Footnotes¶

  1. This is the standard definition in Lean’s core library: Not p unfolds to p → False, so a proof of ¬P is exactly a function turning a proof of P into a proof of False.
